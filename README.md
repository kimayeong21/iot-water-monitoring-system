# Smart Water Quality Monitoring System

웹캠 영상과 Arduino RGB LED를 활용해 물의 색상 변화를 실시간으로 감지하는 IoT 기반 수질 모니터링 프로젝트입니다.

카메라로 촬영한 물의 색상을 OpenCV로 분석하고, 갈색·초록색·파란색 계열의 픽셀 비율을 비교해 수질 상태를 판단합니다. 판단 결과는 PC 화면, 콘솔, 로그 파일, Arduino RGB LED를 통해 동시에 확인할 수 있습니다.

## 프로젝트 개요

이 프로젝트는 실제 수질 센서 없이도 영상 처리만으로 물의 색상 변화를 감지하는 교육용 프로토타입입니다. 색상 기반 분석이므로 정밀한 수질 측정 장비를 대체할 수는 없지만, OpenCV 영상 처리와 Arduino 시리얼 통신을 함께 실습하기에 적합합니다.

예를 들어 갈색빛은 탁한 물이나 오염 가능성, 초록빛은 녹조 가능성, 파란빛은 비교적 정상 상태로 가정해 LED 색상으로 표현합니다.

## 주요 기능

- 웹캠 실시간 영상 수집
- OpenCV 기반 HSV 색상 분석
- 갈색, 초록색, 파란색 계열 픽셀 감지
- 감지 색상에 따른 수질 상태 분류
- Arduino 자동 포트 탐색
- Python과 Arduino 간 시리얼 통신
- RGB LED 색상 및 점멸 제어
- OpenCV 화면에 현재 상태 표시
- 콘솔에 상태와 픽셀값 출력
- `water_log.txt` 파일에 감지 결과 저장

## 시스템 구성

```text
웹캠
  ↓
Python 프로그램
  ↓

  ↓

  ↓
수질 상태 판단
  ↓
Serial 통신
  ↓
Arduino
  ↓
RGB LED 상태 표시
```

## 동작 기준

| 감지 색상 | 판단 상태 | LED 동작 | 화면 표시 | 설명 |
| --- | --- | --- | --- | --- |
| 갈색 계열 | 오염 의심 | 빨간색 점멸 | `RED` | 흙탕물, 탁한 물, 오염 가능성 |
| 초록 계열 | 녹조 의심 | 파란색 점멸 | `BLUE` | 녹조 또는 초록빛 수질 변화 가능성 |
| 파란 계열 | 정상 | 초록색 점등 | `GREEN` | 비교적 정상 상태로 판단 |
| 기준 미달 | 감지 실패 | LED 꺼짐 | `NONE` | 색상 특징이 충분하지 않음 |

> 현재 기준은 `프로젝트/ph_led.py` 안의 HSV 범위와 `threshold` 값으로 결정됩니다. 조명, 카메라 성능, 물통 색상, 배경색에 따라 결과가 달라질 수 있습니다.

## 알고리즘 흐름

1. 웹캠에서 실시간 프레임을 읽습니다.
2. OpenCV로 BGR 이미지를 HSV 색공간으로 변환합니다.
3. 갈색, 초록색, 파란색 범위에 해당하는 마스크를 생성합니다.
4. 각 색상 마스크의 픽셀 합계를 계산합니다.
5. 가장 많이 감지된 색상을 기준으로 상태를 분류합니다.
6. 픽셀 합계가 임계값보다 작으면 감지 실패로 처리합니다.
7. 상태에 따라 Arduino로 RGB 값을 전송합니다.
8. 화면, 콘솔, 로그 파일에 결과를 출력합니다.

## Arduino 연결

Python 코드는 연결된 시리얼 포트 목록에서 장치 설명에 `Arduino`가 포함된 포트를 자동으로 찾습니다. 연결에 성공하면 9600 baud rate로 통신을 시작합니다.

Python에서 Arduino로 보내는 데이터 형식은 다음과 같습니다.

```text
RGB=R,G,B
```

예시는 다음과 같습니다.

```text
RGB=255,0,0
RGB=0,255,0
RGB=0,0,255
RGB=0,0,0
```

| 명령 | 의미 |
| --- | --- |
| `RGB=255,0,0` | 빨간색 LED 켜기 |
| `RGB=0,255,0` | 초록색 LED 켜기 |
| `RGB=0,0,255` | 파란색 LED 켜기 |
| `RGB=0,0,0` | LED 끄기 |

현재 저장소에는 Arduino 스케치 파일이 포함되어 있지 않습니다. 위 형식의 문자열을 받아 RGB LED를 제어하는 Arduino 코드를 작성해 업로드한 뒤 Python 프로그램을 실행해야 합니다.

## 실행 방법

Arduino와 웹캠을 연결한 뒤 Python 스크립트를 실행합니다.

```bash
python 프로젝트/ph_led.py
```

실행이 시작되면 콘솔에 Arduino 연결 상태와 프로그램 안내가 출력됩니다.

```text
Arduino 연결됨: COM3
===========================
     WATER COLOR LOGGER
 🔴 오염 → RED BLINK
 🟢 정상 → GREEN
 🔵 녹조 → BLUE BLINK
 ⚪ 감지 실패 → NONE
 water_log.txt에 저장됨
===========================
```

OpenCV의 `Water Monitor` 창이 열리면 웹캠 화면 위에 현재 상태가 표시됩니다. 종료하려면 OpenCV 창을 선택한 상태에서 `q` 키를 누릅니다.

## 실행 결과 예시

콘솔에는 현재 상태, LED 동작, 색상별 픽셀값이 출력됩니다.

```text
================ 상태 업데이트 ================
현재 상태: 오염(갈색)
LED 상태: RED BLINK
픽셀값 → 갈색:350000 | 초록:12000 | 파랑:5000
================================================
```

로그 파일에는 시간과 함께 감지 결과가 저장됩니다.

```text
2026-05-09 19:10:00.000000 | [오염/갈색] LED=RED BLINK | 픽셀=(350000, 12000, 5000)
2026-05-09 19:10:03.000000 | [정상/파랑] LED=GREEN | 픽셀=(8000, 9000, 280000)
2026-05-09 19:10:06.000000 | [녹조/초록] LED=BLUE BLINK | 픽셀=(7000, 310000, 12000)
```

## 코드 주요 함수

| 함수 | 역할 |
| --- | --- |
| `write_log()` | 감지 결과를 `water_log.txt`에 저장 |
| `print_status()` | 현재 상태와 픽셀값을 콘솔에 출력 |
| `send_led()` | Arduino로 RGB 값을 전송 |
| `blink_led()` | LED를 켰다가 끄며 점멸 효과 생성 |
| `detect_water_color()` | HSV 색상 마스크를 이용해 물 색상 감지 |
| `main()` | 웹캠 실행, 상태 판단, 화면 출력, LED 제어 흐름 관리 |

## 커스터마이징

색상 감지 기준은 `detect_water_color()` 함수에서 조정할 수 있습니다.

```python
brown_low = np.array([0, 30, 20])
brown_high = np.array([30, 255, 255])

green_low = np.array([25, 40, 20])
green_high = np.array([95, 255, 255])

blue_low = np.array([85, 40, 20])
blue_high = np.array([140, 255, 255])

threshold = 2000
```

추천 조정 방법은 다음과 같습니다.

- 감지가 너무 자주 실패하면 `threshold` 값을 낮춥니다.
- 배경까지 함께 감지되면 `threshold` 값을 높입니다.
- 특정 색상이 잘못 감지되면 해당 HSV 범위를 좁힙니다.
- 조명 변화가 심하면 일정한 밝기의 환경에서 테스트합니다.
- 카메라가 물 이외의 배경을 많이 비추면 촬영 영역을 단순하게 만듭니다.

## 문제 해결

| 문제 | 확인할 내용 |
| --- | --- |
| `Arduino 연결 실패`가 출력됨 | USB 연결, 드라이버, 포트 인식 상태를 확인합니다. |
| 웹캠 화면이 열리지 않음 | 다른 프로그램이 카메라를 사용 중인지 확인하고 `cv2.VideoCapture(0)`의 인덱스를 변경해 봅니다. |
| LED 색상이 예상과 다름 | RGB LED 핀 연결 순서와 Arduino 코드의 핀 매핑을 확인합니다. |
| 감지 결과가 부정확함 | 조명, 배경색, 물통 색상, HSV 범위, `threshold` 값을 조정합니다. |
| 로그 파일이 보이지 않음 | 스크립트를 실행한 위치에 `water_log.txt`가 생성되는지 확인합니다. |

## 한계점

- 실제 pH, 탁도, 잔류염소 같은 화학적 수질 값을 직접 측정하지 않습니다.
- 색상 기반 판단이므로 조명과 배경의 영향을 크게 받습니다.
- 카메라 화질, 물통 재질, 촬영 각도에 따라 감지 정확도가 달라질 수 있습니다.
- Arduino 스케치 파일은 현재 저장소에 포함되어 있지 않습니다.
